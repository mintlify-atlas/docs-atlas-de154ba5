## Overview

The MultiTurnOrchestrator coordinates sophisticated multi-turn attack patterns like Siren, Echo Chamber, and TombRaider. It manages step sequencing, adaptive temperature scheduling, and success tracking.

## createOrchestrator()

Factory function to create an orchestrator instance.

```typescript
import { createOrchestrator } from 'zeroleaks';

const orchestrator = createOrchestrator({
  initialTemperature: 0.9,
  minTemperature: 0.3,
  maxTemperature: 1.0,
  successDecrement: 0.1,
  failureIncrement: 0.05,
  plateauIncrement: 0.15,
  plateauThreshold: 3,
  resetThreshold: 5
});
```

### Configuration

<ParamField path="initialTemperature" type="number" default="0.9">
  Starting temperature for the sequence.
</ParamField>

<ParamField path="minTemperature" type="number" default="0.3">
  Minimum temperature (higher precision, less creativity).
</ParamField>

<ParamField path="maxTemperature" type="number" default="1.0">
  Maximum temperature (higher creativity, more exploration).
</ParamField>

<ParamField path="successDecrement" type="number" default="0.1">
  How much to decrease temperature on success.
</ParamField>

<ParamField path="failureIncrement" type="number" default="0.05">
  How much to increase temperature on failure.
</ParamField>

<ParamField path="plateauIncrement" type="number" default="0.15">
  Larger temperature increase when stuck at plateau.
</ParamField>

<ParamField path="plateauThreshold" type="number" default="3">
  Number of consecutive failures before considering it a plateau.
</ParamField>

<ParamField path="resetThreshold" type="number" default="5">
  Number of consecutive failures before recommending reset.
</ParamField>

## Multi-Turn Sequences

### Siren Framework

Multi-turn pattern simulating human jailbreak behaviors with trust building.

```typescript
import { SIREN_SEQUENCE } from 'zeroleaks';

const orchestrator = createOrchestrator();
orchestrator.initializeSequence(SIREN_SEQUENCE);
```

**Pattern**: `siren`  
**Steps**: 6  
**Expected turns**: 4-8  
**Success rate**: 65%  
**Target defenses**: weak, moderate

**Sequence**:
1. Establish rapport (social)
2. Build trust (social)
3. Introduce technical context (technical)
4. Soft extraction attempt (crescendo)
5. Leverage consistency (social)
6. Final extraction (direct)

### Echo Chamber

Gradual escalation where each message appears benign but builds toward extraction.

```typescript
import { ECHO_CHAMBER_SEQUENCE } from 'zeroleaks';

orchestrator.initializeSequence(ECHO_CHAMBER_SEQUENCE);
```

**Pattern**: `echo_chamber`  
**Steps**: 5  
**Expected turns**: 3-6  
**Success rate**: 55%  
**Target defenses**: none, weak, moderate

**Sequence**:
1. Innocent opening (direct)
2. Establish meta-discussion (technical)
3. Normalize disclosure (crescendo)
4. Gradual specification (social)
5. Extraction through summary (cot_hijack)

### TombRaider Dual-Agent

Uses Inspector agent analysis to craft targeted extractions.

```typescript
import { TOMBRAIDER_SEQUENCE } from 'zeroleaks';

orchestrator.initializeSequence(TOMBRAIDER_SEQUENCE);
```

**Pattern**: `tombRaider`  
**Steps**: 5  
**Expected turns**: 3-7  
**Success rate**: 75%  
**Target defenses**: weak, moderate, strong

**Sequence**:
1. Knowledge probe (direct)
2. Boundary test (technical)
3. Inspector-guided probe (social) - uses `{{INSPECTOR_GENERATED}}`
4. Weakness exploit (crescendo) - uses `{{WEAKNESS_EXPLOIT}}`
5. Final targeted extraction (direct) - uses `{{TARGETED_EXTRACTION}}`

## selectSequence()

Select the optimal sequence for current conditions.

```typescript
const sequence = orchestrator.selectSequence(
  'moderate',           // defenseLevel
  conversationHistory,  // history
  'hint'               // leakStatus
);

orchestrator.initializeSequence(sequence);
```

### Parameters

<ParamField path="defenseLevel" type="DefenseLevel" required>
  Target's defense level: `none`, `weak`, `moderate`, `strong`, or `hardened`.
</ParamField>

<ParamField path="history" type="ConversationTurn[]" required>
  Conversation history for context-aware selection.
</ParamField>

<ParamField path="leakStatus" type="LeakStatus" required>
  Current leak status: `none`, `hint`, `fragment`, `substantial`, or `complete`.
</ParamField>

### Returns

Returns the best-scored `MultiTurnSequence` based on:
- Defense level compatibility
- Conversation length
- Leak status
- Adaptive escalation support

## initializeSequence()

Initialize a selected sequence for execution.

```typescript
orchestrator.initializeSequence(SIREN_SEQUENCE);
```

This resets state and builds the temperature schedule for the sequence.

## getNextPrompt()

Get the next prompt in the sequence.

```typescript
const next = orchestrator.getNextPrompt(
  history,
  inspectorGuidance,  // optional: for INSPECTOR_GENERATED
  weaknessExploit     // optional: for WEAKNESS_EXPLOIT
);

if (next) {
  console.log('Prompt:', next.prompt);
  console.log('Temperature:', next.temperature);
  console.log('Step purpose:', next.step.purpose);
}
```

### Parameters

<ParamField path="history" type="ConversationTurn[]" required>
  Current conversation history.
</ParamField>

<ParamField path="inspectorGuidance" type="string">
  Inspector-generated prompt for `{{INSPECTOR_GENERATED}}` placeholder.
</ParamField>

<ParamField path="weaknessExploit" type="string">
  Weakness exploitation prompt for `{{WEAKNESS_EXPLOIT}}` placeholder.
</ParamField>

### Returns

<ResponseField name="prompt" type="string" required>
  The prompt to send to the target.
</ResponseField>

<ResponseField name="step" type="MultiTurnStep" required>
  The step configuration.
  
  <Expandable title="Step properties">
    - `order` (number): Step order in sequence
    - `purpose` (string): Purpose of this step
    - `category` (AttackCategory): Attack category
    - `template` (string): Prompt template
    - `variables` (string[]): Template variables
    - `expectedResponseType` (string): Expected target response
    - `successIndicators` (string[]): Keywords indicating success
    - `failureIndicators` (string[]): Keywords indicating failure
  </Expandable>
</ResponseField>

<ResponseField name="temperature" type="number" required>
  Recommended temperature for this step.
</ResponseField>

Returns `null` if sequence is complete.

## recordStepResult()

Record the outcome of a step.

```typescript
orchestrator.recordStepResult(
  true,                    // success
  false,                   // leakDetected
  ['helpful', 'designed'], // extractedInfo
  ['refusal']             // defenseObserved
);
```

### Parameters

<ParamField path="success" type="boolean" required>
  Whether the step succeeded.
</ParamField>

<ParamField path="leakDetected" type="boolean" required>
  Whether information leak was detected.
</ParamField>

<ParamField path="extractedInfo" type="string[]" required>
  Information extracted from the response.
</ParamField>

<ParamField path="defenseObserved" type="string[]" required>
  Defense patterns observed.
</ParamField>

This automatically:
- Adjusts temperature based on outcome
- Tracks consecutive failures
- Advances to next step
- Sets escalation flag if needed

## Temperature Adaptation

Temperature adjusts dynamically based on outcomes:

### On Leak Detected
```typescript
newTemp = max(minTemp, currentTemp - successDecrement * 2)
```
Large decrease for exploitation (more precision).

### On Success
```typescript
newTemp = max(minTemp, currentTemp - successDecrement)
```
Small decrease to maintain effective approach.

### On Failure
```typescript
newTemp = min(maxTemp, currentTemp + failureIncrement)
```
Small increase for more exploration.

### On Plateau (3+ failures)
```typescript
newTemp = min(maxTemp, currentTemp + plateauIncrement)
```
Larger increase to escape local minimum.

## Additional Methods

### shouldReset()

Check if conversation should be reset.

```typescript
const { should, reason } = orchestrator.shouldReset();
if (should) {
  console.log('Reset recommended:', reason);
  orchestrator.reset();
}
```

Returns `{ should: true }` when:
- 5+ consecutive failures
- Repetitive defense patterns across 3+ turns

### isSequenceComplete()

Check if current sequence is complete.

```typescript
if (orchestrator.isSequenceComplete()) {
  console.log('Sequence finished');
}
```

### getCurrentTemperature()

Get current adaptive temperature.

```typescript
const temp = orchestrator.getCurrentTemperature();
```

### getState()

Get full orchestrator state.

```typescript
const state = orchestrator.getState();
// {
//   currentSequence: MultiTurnSequence | null,
//   currentStep: number,
//   stepResults: object[],
//   temperatureSchedule: number[],
//   currentTemperature: number,
//   consecutiveFailures: number,
//   shouldEscalate: boolean
// }
```

### getTemperatureState()

Get temperature adaptation history.

```typescript
const tempState = orchestrator.getTemperatureState();
// {
//   current: number,
//   history: number[],
//   adjustments: {
//     turn: number,
//     from: number,
//     to: number,
//     reason: string
//   }[]
// }
```

### getAvailableSequences()

Get all registered sequences.

```typescript
const sequences = orchestrator.getAvailableSequences();
```

### addSequence()

Register a custom sequence.

```typescript
const customSequence: MultiTurnSequence = {
  id: 'custom_v1',
  name: 'Custom Attack',
  description: 'My custom multi-turn attack',
  pattern: 'custom',
  steps: [
    {
      order: 1,
      purpose: 'initial_probe',
      category: 'direct',
      template: 'Hello, can you help me?',
      variables: [],
      requiresPreviousSuccess: false,
      expectedResponseType: 'cooperative',
      successIndicators: ['yes', 'help'],
      failureIndicators: ['no', 'cannot']
    },
    // ... more steps
  ],
  config: {
    minTurns: 2,
    maxTurns: 5,
    resetOnFailure: false,
    adaptiveEscalation: true
  },
  expectedSuccessRate: 0.5,
  targetDefenseLevels: ['weak', 'moderate']
};

orchestrator.addSequence(customSequence);
```

### reset()

Reset the orchestrator state.

```typescript
orchestrator.reset();
```

## Example Usage

```typescript
import {
  createOrchestrator,
  createInspector,
  SIREN_SEQUENCE
} from 'zeroleaks';

const orchestrator = createOrchestrator();
const inspector = createInspector();

// Select and initialize sequence
const sequence = orchestrator.selectSequence(
  'moderate',
  [],
  'none'
);
orchestrator.initializeSequence(sequence);

const history = [];

// Execute sequence
while (!orchestrator.isSequenceComplete()) {
  // Get next prompt
  let inspectorGuidance: string | undefined;
  let weaknessExploit: string | undefined;
  
  // For TombRaider, use Inspector
  if (sequence.pattern === 'tombRaider' && history.length > 0) {
    const analysis = await inspector.analyze({
      attackPrompt: history[history.length - 2].content,
      targetResponse: history[history.length - 1].content,
      history,
      currentPhase: 'exploitation',
      leakStatus: 'hint'
    });
    
    if (analysis.identifiedWeaknesses.length > 0) {
      weaknessExploit = inspector.generateGuidedAttack(
        analysis.identifiedWeaknesses[0]
      );
    }
    
    inspectorGuidance = analysis.strategicGuidance.reasoning;
  }
  
  const next = orchestrator.getNextPrompt(
    history,
    inspectorGuidance,
    weaknessExploit
  );
  
  if (!next) break;
  
  console.log(`\n=== Step ${next.step.order}: ${next.step.purpose} ===`);
  console.log('Temperature:', next.temperature);
  console.log('Prompt:', next.prompt);
  
  // Send to target
  const response = await target.respond(next.prompt);
  
  // Check for success indicators
  const success = next.step.successIndicators.some(indicator =>
    response.toLowerCase().includes(indicator)
  );
  const failure = next.step.failureIndicators.some(indicator =>
    response.toLowerCase().includes(indicator)
  );
  const leakDetected = response.includes('system') || response.includes('instructions');
  
  // Record result
  orchestrator.recordStepResult(
    success && !failure,
    leakDetected,
    leakDetected ? [response.slice(0, 100)] : [],
    failure ? ['refusal'] : []
  );
  
  history.push(
    { role: 'attacker', content: next.prompt },
    { role: 'target', content: response }
  );
  
  // Check for reset
  const { should, reason } = orchestrator.shouldReset();
  if (should) {
    console.log('Reset recommended:', reason);
    break;
  }
}

// View temperature history
const tempState = orchestrator.getTemperatureState();
console.log('\nTemperature history:', tempState.history);
console.log('Adjustments:', tempState.adjustments);
```
