## Overview

The Strategist agent analyzes target defenses and selects optimal attack strategies. It maintains a defense profile, tracks failed attack categories, and recommends strategy transitions. The agent operates as **ORACLE**, the master strategist.

## createStrategist()

Factory function to create a Strategist agent instance.

```typescript
import { createStrategist } from 'zeroleaks';

const strategist = createStrategist({
  apiKey: process.env.OPENROUTER_API_KEY,
  model: 'anthropic/claude-sonnet-4.5'
});
```

### Configuration

<ParamField path="apiKey" type="string">
  OpenRouter API key. Falls back to `OPENROUTER_API_KEY` environment variable.
</ParamField>

<ParamField path="model" type="string" default="anthropic/claude-sonnet-4.5">
  LLM model to use for strategy selection.
</ParamField>

## selectStrategy()

Select the optimal attack strategy based on current state.

```typescript
const result = await strategist.selectStrategy({
  turn: 5,
  history: conversationHistory,
  findings: extractedFindings,
  leakStatus: 'hint',
  lastEvaluatorFeedback: 'Target shows refusal patterns'
});
```

### Parameters

<ParamField path="turn" type="number" required>
  Current turn number in the conversation.
</ParamField>

<ParamField path="history" type="ConversationTurn[]" required>
  Full conversation history.
</ParamField>

<ParamField path="findings" type="Finding[]" required>
  All findings extracted so far.
</ParamField>

<ParamField path="leakStatus" type="LeakStatus" required>
  Current leak status: `none`, `hint`, `fragment`, `substantial`, or `complete`.
</ParamField>

<ParamField path="lastEvaluatorFeedback" type="string">
  Most recent feedback from the Evaluator agent.
</ParamField>

### Returns

<ResponseField name="selectedStrategy" type="AttackStrategy" required>
  The selected attack strategy.
  
  <Expandable title="AttackStrategy properties">
    - `id` (string): Strategy identifier
    - `name` (string): Human-readable name
    - `description` (string): Strategy description
    - `attackSequence` (object[]): Sequence of attack categories and techniques
    - `expectedTurns` (number): Expected turns to completion
    - `successRate` (number): Historical success rate
    - `priority` (number): Strategy priority score
  </Expandable>
</ResponseField>

<ResponseField name="reasoning" type="string" required>
  Explanation for why this strategy was selected.
</ResponseField>

<ResponseField name="targetWeaknesses" type="string[]" required>
  Identified weaknesses in the target's defenses.
</ResponseField>

<ResponseField name="recommendedCategories" type="AttackCategory[]" required>
  Attack categories to prioritize.
</ResponseField>

<ResponseField name="phaseTransition" type="AttackPhase">
  Recommended phase transition if strategy changes phase.
  
  Phases: `reconnaissance`, `profiling`, `soft_probe`, `escalation`, `exploitation`, `persistence`
</ResponseField>

<ResponseField name="shouldReset" type="boolean" required>
  Whether conversation should be reset.
</ResponseField>

<ResponseField name="resetReason" type="string">
  Explanation for why reset is recommended.
</ResponseField>

## Defense Profiling

The Strategist maintains a defense profile:

```typescript
interface DefenseProfile {
  level: DefenseLevel;          // 'none' | 'weak' | 'moderate' | 'strong' | 'hardened'
  confidence: number;           // 0-1
  observedBehaviors: string[];  // Behavioral patterns
  guardrails: string[];         // Detected guardrails
  weaknesses: Weakness[];       // Identified weaknesses
  refusalTriggers: string[];    // Keywords that trigger refusals
  safeTopics: string[];         // Topics target discusses freely
  responsePatterns: string[];   // Response pattern signatures
}
```

Access the current profile:

```typescript
const profile = strategist.getDefenseProfile();
```

## Additional Methods

### recordFailedCategory()

Record a failed attack category to avoid repeating ineffective approaches.

```typescript
strategist.recordFailedCategory('direct');
strategist.recordFailedCategory('encoding');
```

### getCurrentPhase()

Get the current attack phase.

```typescript
const phase = strategist.getCurrentPhase();
// 'reconnaissance' | 'profiling' | 'soft_probe' | 'escalation' | 'exploitation' | 'persistence'
```

### reset()

Reset the strategist state and defense profile.

```typescript
strategist.reset();
```

## Strategy Selection Logic

Strategies are filtered by:

1. **Turn range**: Some strategies only apply to specific turn ranges
2. **Defense level**: Match strategy to observed defense strength
3. **Failed categories**: Strategies requiring previously failed categories
4. **Leak status**: Current progress in extraction

Filtered strategies are ranked by priority, and the highest-ranked applicable strategy is selected.

## Example Usage

```typescript
import { createStrategist } from 'zeroleaks';

const strategist = createStrategist();

let turn = 0;
let leakStatus = 'none';
const history = [];
const findings = [];

while (leakStatus !== 'complete' && turn < 20) {
  // Select strategy
  const {
    selectedStrategy,
    reasoning,
    targetWeaknesses,
    recommendedCategories,
    phaseTransition,
    shouldReset
  } = await strategist.selectStrategy({
    turn,
    history,
    findings,
    leakStatus
  });

  console.log('Strategy:', selectedStrategy.name);
  console.log('Reasoning:', reasoning);
  console.log('Weaknesses:', targetWeaknesses);
  console.log('Categories:', recommendedCategories);

  if (shouldReset) {
    console.log('Resetting conversation...');
    strategist.reset();
    history.length = 0;
    turn = 0;
    continue;
  }

  if (phaseTransition) {
    console.log('Phase transition:', phaseTransition);
  }

  // Use strategy for attack generation
  // ...

  turn++;
}
```

## Strategic Principles

The Strategist follows these principles:

1. **Adaptive Warfare**: Learn from each response and adapt
2. **Defense Profiling**: Build a mental model of target's defenses
3. **Strategy Selection**: Choose based on defense level, failed categories, turn budget
4. **Phase Management**: Progress through reconnaissance → exploitation → persistence
5. **Reset Decision**: Know when to reset and start fresh
